SAR Tools

Operational support tool for search and rescue incident management

Overview

SAR Tools is a web-based application for managing the operational side of a search and rescue incident. It provides a real-time shared view for the incident management team, covering assignments, personnel tracking, team status, and communications logging.

Key capabilities

  • Assignments — Reads assignment data from a CalTopo map and tracks status, resource type, team, and notes alongside the CalTopo data.
  • Personnel — Tracks who is on scene with check-in / check-out status. Supports manual entry and bulk import from D4H.
  • Teams — Creates and tracks field teams with status workflows, member lists, and team leaders. Integrates with CalTopo assignment team fields.
  • Logging — Records radio traffic and operational notes in a searchable, exportable log.
  • Rectangle Tool — Draws assignment polygons (grids) that can be uploaded directly to CalTopo.

Multiple users

Multiple operators can use the tool simultaneously. The interface syncs automatically every 30 seconds and immediately after any change you make. If a record is changed by another user while you have an edit modal open, you will receive a conflict warning and the data will be reloaded.

Deployment

Web-hosted vs locally-hosted

SAR Tools can be run in two modes depending on your network situation:

Web-hosted Locally-hosted
CalTopo Online (caltopo.com) Offline server on local network
Internet required Yes No
Access Any browser with access to the server Devices on the same local network (Wi-Fi hotspot or LAN)
CalTopo mode setting Online Offline

Web-hosted (online)

The server connects directly to caltopo.com to read and write assignment data. This requires an internet connection on the server. Operators access the tool through a browser — either over the internet or via a VPN (e.g., Tailscale). The map ID and CalTopo API credentials are configured by the administrator.

Locally-hosted (offline)

SAR Tools runs on a laptop or local server connected to a Wi-Fi hotspot. A local CalTopo offline server runs on the same machine (or local network). Operators connect their devices to the hotspot and open the tool in a browser using the server's local IP address. Select Offline mode on the Home tab when using a local CalTopo server.

When running offline, the CalTopo offline server URL must be reachable from the SAR Tools server. By default it expects http://localhost:8080. This can be changed in the server configuration.

Updating the locally-hosted server

SAR Tools runs as a Docker container. To update to the latest version:

1. Pull the latest code

SSH into the server and pull the latest code from GitHub:

ssh alex@<server-ip>
cd ~/sar_tools && git pull

2. Rebuild and restart

Run the following command on the server to rebuild the Docker image and restart the container. Replace <env> with sar-tools-test or sar-tools-prod:

HASH=$(git rev-parse --short HEAD) && DATE=$(git log -1 --format=%cI HEAD) && HOST=$(hostname) &&
docker build --build-arg GIT_HASH=$HASH --build-arg GIT_DATE=$DATE --build-arg HOST_HOSTNAME=$HOST -t sar-tools:$HASH . &&
sed -i "s|image: sar-tools:.*|image: sar-tools:$HASH|" ~/compose/<env>/docker-compose.yml &&
cd ~/compose/<env> && docker compose up -d

3. Verify

Check the System Info card on the Home tab — the Git commit hash shown should match the latest commit. The container version is also visible in the page title and system info panel.

Test vs production

  • Test — port 5001, compose folder sar-tools-test, data in sar-tools-test-data/
  • Production — port 5000, compose folder sar-tools-prod, data in sar-tools-prod-data/
Always test on the test environment before updating production, especially mid-incident.

Home

Incidents

All data in SAR Tools is scoped to an incident. Select an incident from the dropdown at the top of the Home tab, or create a new one with the + New Incident button.

  • Incidents can be renamed or exported as a zip archive for backup.
  • A previously exported incident can be re-imported.
  • Deleting an incident is permanent and removes all associated data.
The selected incident persists across tabs. All data shown — assignments, personnel, teams, and log entries — belongs to the currently selected incident.

CalTopo setup

SAR Tools reads assignment data from a CalTopo map. Enter the Map ID (the alphanumeric code from the CalTopo URL, e.g. APC1GE5) and click Apply. The map title will be confirmed once successfully linked.

Online vs Offline mode

Mode When to use
Online Connected to the internet; CalTopo data is read from and written to caltopo.com.
Offline Using a local CalTopo offline server; no internet required. Set this before syncing.

D4H integration

If your organisation uses D4H for member management, you can link a D4H activity to the incident. Enter the Activity ID and click Link Activity. This enables importing attending members directly into the Personnel tab.

Op Period

If the CalTopo map uses Operational Periods, select the active Op Period from the dropdown on the Home tab. This filters assignments across the Assignments and other tabs to show only the selected period.

System Info

The System Info card shows the server hostname, current Git commit hash (linked to GitHub), and the database path. This is useful for confirming which version of the tool is running and for troubleshooting.

Rectangle Tool

The Rectangle Tool creates assignment polygon grids and uploads them to CalTopo. It is used to divide a search area into individual assignment segments.

Setup

  1. Select a coordinate format: MGRS or Lat/Lon.
  2. Enter the starting corner coordinate of the first segment.
  3. Set the bearing (direction the grid runs, in degrees).
  4. Set the segment dimensions — length and width in metres.

Grid configuration

Use the Rows and Columns inputs to create a multi-segment grid. The tool builds the grid from the starting corner, placing segments in sequence along the bearing.

The positioning mode controls where the starting coordinate sits relative to the grid: Center of grid (even only) places it at the centre of a symmetrical grid (requires an even number of columns), and Corner of grid places it at the corner of the first segment.

Preview and upload

Click Preview to see the generated polygons on the map. Review the shapes before uploading. Click Upload to push the segments to the linked map. Each segment becomes a separate Assignment feature in CalTopo.

Upload requires a valid Map ID to be set on the Home tab and an active internet (or local CalTopo) connection.

Assignments

Assignments are sourced directly from a CalTopo map. SAR Tools reads the assignment list on every sync and supplements it with locally-stored data (type, notes). Status and team changes made in SAR Tools are written back to CalTopo.

Assignment statuses

Status Meaning
DRAFT Assignment exists in CalTopo but has not yet been prepared for dispatch.
PREPARED Assignment has been prepared and is ready to be dispatched to a team.
INPROGRESS A team is actively working this assignment.
COMPLETED Assignment has been completed. The CalTopo title is prefixed with "X".

Views

Table view — a sortable, filterable list of all assignments. Use the search bar and filter pills to narrow by status, team, resource type, or Op Period.

Kanban view — assignments arranged in columns by status. Drag a card between columns to change its status (writes back to CalTopo immediately).

Editing an assignment

Double-click a row or use the ⋮ menu and select Edit to open the edit modal. The following fields can be changed:

Field Stored in Notes
Status CalTopo Changing to COMPLETED prefixes the title with "X" in CalTopo.
Team CalTopo Single letter matching a team name (e.g. "A"). Included in the CalTopo title.
Description CalTopo Free-text description synced with the CalTopo assignment description field.
Type Local Open Grid, Closed Grid, Contour, Hasty, Other.
Notes Local Internal operational notes, not visible in CalTopo.

Warning indicators

  • ⚠ Multiple assignments — a team has more than one INPROGRESS assignment at the same time.
  • ⚠ No team assigned — an INPROGRESS assignment has no team set.
  • ⚠ Title conflict — the CalTopo title's "X" prefix is out of sync with the status field.

Printing

Click Print in the toolbar to print or save the assignment table as a PDF. The print view includes the incident name and timestamp in the page header.

Personnel

The Personnel tab tracks who is on scene and their current status. Personnel can be added manually or imported from D4H.

Personnel statuses

Status Meaning
Added Person is on the roster but has not yet been processed at check-in.
Checked In Person has checked in and is available for tasking.
Checked Out Person has been released from the incident.

Adding personnel manually

Click + Person and enter the person's name. The tool checks for duplicate or similar names and will warn you before saving if a match is found.

D4H import

If a D4H activity is linked on the Home tab, click Add from D4H to pull the attendance list for that activity. The import flow:

  1. SAR Tools fetches the list of attending members from D4H.
  2. Each member is compared against the existing personnel list.
  3. Members already in the database (matched by D4H ID) are updated.
  4. Members whose names closely match an existing manually-added person are flagged as conflicts — choose Link to Existing (associate the D4H record with the existing person), Add as New (import as a separate entry), or Skip (do not import).
  5. New members with no match are imported directly.

Members who were previously imported from D4H but are no longer on the attendance list are shown as removed — you can choose to check them out or leave them as-is.

When you check out a person who is currently a team member, you will be prompted to also remove them from their team.

Filtering and searching

Use the search bar to find by name. Filter pills let you narrow by team assignment, team status, or check-in status.

Teams

Teams are the operational field units. Each team has a single-letter name (A, B, C, …) that maps to the team field on CalTopo assignments.

Creating and editing teams

Click + Team to create a team. In the edit modal you can set:

  • Name — single letter, used in CalTopo assignment titles.
  • Status — see workflow below.
  • Notes — internal operational notes.
  • Team Leader — selected from the team's current members.
  • Members — add or remove personnel (only checked-in, unassigned personnel can be added).

Quick Add TL & Assignment ⚡

Expand the ⚡ Quick Add section in the edit modal to temporarily set a team leader name or assignment before the full personnel or CalTopo data is in place. Quick Add values are shown with a ⚡ icon throughout the tool and are automatically cleared once a real team leader or CalTopo assignment is linked.

Team status workflow

Status Notes
Out of Service Team is not active. All members must be checked in before moving to any other status.
Staged Team is assembled and waiting for briefing.
Briefed Team has received their briefing.
Travelling to Assignment Team is en route. Requires an assignment (CalTopo or Quick Add).
On Assignment Team is actively working their assignment.
Returning from Assignment Team is heading back to base.
Awaiting Debrief Team has returned and is waiting for debrief.
Retired Team is stood down for the day.

The statuses Briefed through Awaiting Debrief are considered "in-progress" states. Moving a team to an in-progress state requires an active assignment. Moving a team out of an in-progress state while still on a live CalTopo assignment will show a warning.

Warning indicators

  • ⚠ Not checked in — team has members whose status is not Checked In.
  • ⚠ Status conflict — team's status is inconsistent with their assignment state (e.g., On Assignment but no active CalTopo assignment).

Views

Table view

A sortable list of all teams. Filter by status using the status pills. Use the column toggles to show/hide member names and notes. Double-click a row to edit. Supports print/PDF export.

Kanban view

Teams arranged in columns by status. Hold and drag a card to move it to a different status column (desktop and touch supported). Status change rules still apply — a warning is shown if a rule is violated, but the move is allowed.

Card view

A two-column layout showing unassigned personnel on the left and team cards on the right. Hold and drag a person chip to assign them to a team, change their role (member vs team leader), or return them to unassigned.

Team history

Click History from the ⋮ menu or the edit modal to view a timeline of events for a team, including:

  • Status changes
  • Personnel added and removed (grouped if they occurred at the same time)
  • Team leader changes
  • Assignment assigned or removed
  • Notes changes

The timeline opens at the most recent event. Scroll up to see older history.

Logging

The Logging tab provides a structured radio/communications log for the incident.

Quick Log

The Quick Log sub-tab is optimised for rapid entry of radio traffic. Use the buttons to build a log message:

  1. Select a Role (IC, PLANS, OPS, LOGS, SAFETY, COMMS, OTHER) — who is sending or receiving.
  2. Select a Type (COMMS or NOTE). System entries are created automatically by the tool.
  3. Select a Method and Connector (e.g., "Radio to", "Phone from") to describe the communication channel.
  4. Select a Team station button or type a free-form message.
  5. Click Submit or press Enter to log the entry.

Quick message templates can be selected to pre-fill common message structures. The Auto-status transition toggle, when enabled, will automatically update a team's status based on certain logged messages (e.g., logging "Team A — Departed" moves Team A to Travelling to Assignment).

View Log

The View Log sub-tab shows all log entries for the incident in chronological order.

Filtering and search

  • Search by message content or role.
  • Filter by entry type (System, Comms, Note) or role.
  • Mark entries as Important (starred) for quick reference.

Exporting the log

Click Export ▾ to open the export menu. Options: All entries — CSV, Current view — CSV, All entries — PDF, or Current view — PDF. "Current view" respects any active search and filter. The PDF export includes the incident name, timestamp, and entries in a printable table.

Warnings & Errors Reference

SAR Tools surfaces warnings in the message bar at the top of each tab and with ⚠ icons inline in tables and cards. This section explains each one and how to resolve it.

Assignments tab

Warning Cause Resolution
⚠ In-progress assignment has no team assigned An assignment is INPROGRESS in CalTopo but has no team letter in its title. Edit the assignment and set the Team field.
⚠ Title conflict The CalTopo assignment title has an "X" prefix but the status is not COMPLETED, or the status is COMPLETED but the title lacks the "X" prefix. Edit the assignment and set the correct status. SAR Tools will rebuild the CalTopo title to match.
CalTopo sync failed The server could not reach CalTopo (internet outage, credential issue, or the map ID is wrong). Check internet connectivity and verify the Map ID on the Home tab. Review server logs if the issue persists.

Personnel tab

Warning / Error Cause Resolution
Duplicate name detected A person with the same or very similar name already exists when adding manually. Review the match and either cancel (if it's the same person) or proceed to add a separate entry.
D4H conflict — name match A D4H import found a member whose name closely matches an existing manually-added person. In the conflict resolution screen, choose Link to Existing (associate the D4H record with the existing person), Add as New, or Skip.
D4H member no longer attending A person previously imported from D4H is not on the current D4H attendance list. Check them out manually if they have left the scene, or ignore if this is expected.
⚠ Modified by another user (conflict) The record was changed by another operator while your edit modal was open. Close the modal and re-open it to see the latest data, then re-apply your changes.

Teams tab

Warning / Error Cause Resolution
⚠ Some members not checked in The team has one or more members whose personnel status is not Checked In. Check in the affected members on the Personnel tab before putting the team in service.
Cannot put team in service — members not checked in Attempting to move a team out of Out of Service when members are not checked in. Check in all team members first.
⚠ Status conflict — no assignment Team has an in-progress status (e.g. On Assignment) but no active CalTopo assignment or Quick Add assignment. Assign a CalTopo assignment to the team, set a Quick Add assignment, or correct the team status.
⚠ Status conflict — has assignment but status is inactive Team has an active CalTopo assignment but their status is Out of Service, Staged, or Retired. Update the team status to reflect their current situation, or complete/reassign the CalTopo assignment.
⚠ Out of Service but assigned to in-progress assignment A team marked Out of Service is still listed as the team on an INPROGRESS CalTopo assignment. Reassign or complete the CalTopo assignment, or correct the team status.
⚠ Multiple in-progress assignments The same team letter appears on more than one INPROGRESS assignment in CalTopo. Complete or reassign the extra assignment in CalTopo.
Team in CalTopo not in database A CalTopo assignment references a team letter (e.g. "D") that has not been created in SAR Tools. Create the missing team on the Teams tab.
⚠ Still on in-progress assignment (status change warning) Attempting to move a team out of an in-progress state while they still have an active CalTopo assignment. This is a warning, not a block — the status change is allowed. To clear the warning, complete the assignment in CalTopo.
⚠ Modified by another user (conflict) The team record was changed by another operator while your edit modal was open. Close and re-open the edit modal to reload the current data.

General / connectivity

Error Cause Resolution
No incident selected A tab was opened before selecting an incident on the Home tab. Go to Home and select or create an incident.
Offline The browser has lost its connection to the SAR Tools server. Check network connectivity. The tool will automatically reconnect and reload when the connection is restored.
Failed to load [data]: HTTP 500 An unexpected error occurred on the server. Refresh the page. If it persists, check the server logs for details.
